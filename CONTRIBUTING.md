# Contributing to Bank Support

Thanks for taking the time to contribute to Bank Support.

This repository is the public support and feedback channel for the Bank education platform:

👉 https://bank.easyitlab.tech/

The source code is stored in a private repository.  
This repository is used for:
- issue reporting
- feature requests
- questions and discussions
- access / billing / provisioning requests

---

## Before You Create an Issue

Please make sure that:

- you checked the existing Issues
- you checked the Discussions section
- your report is related to the Bank platform
- you do not include sensitive information

### Do NOT share:
- passwords
- tokens
- secret keys
- private URLs
- personal data
- payment details
- internal credentials

If your report contains sensitive or security-related information, do not post it publicly.

---

## What to Use

### Use an Issue for:
- bug reports
- access problems
- billing / provisioning problems
- feature requests with a clear expected result

### Use a Discussion for:
- questions
- ideas
- feedback
- open-ended conversations

---

## Writing a Good Bug Report

A good bug report should include:

- a short summary
- steps to reproduce
- expected behavior
- actual behavior
- environment details
- screenshots or logs if available
- links only if they are safe to share publicly

Example:

1. Open the platform
2. Go to the student cabinet
3. Click on "Generate entities"
4. Receive an error

Expected:
The entities should be generated successfully.

Actual:
The page shows a 500 error.

---

## Writing a Good Feature Request

A good feature request should include:

- what you want to improve
- why it is useful
- what problem it solves
- possible expected behavior

Please keep feature requests practical and specific.

---

## Access, Billing, and Provisioning Requests

If your issue is related to:
- login
- expired access
- wrong permissions
- payment completed but access not activated
- subscription / trial / manual access

please use the appropriate issue form.

Do not post sensitive payment data publicly.

---

## Issue Triage

Submitted issues may be:
- labeled
- clarified
- marked as duplicate
- moved to Discussions
- closed if incomplete or unrelated

If additional information is needed, please respond in the issue thread.

---

## Development Workflow

When a valid issue requires implementation:

1. The issue is triaged
2. A linked development branch is created in the private repository
3. Changes are implemented
4. The work is linked back to the support issue
5. The issue is closed after merge

Examples of internal naming:

- `fix/support-123-login-timeout`
- `feature/support-145-export-csv`

---

## Pull Requests

Pull requests are not expected from external users for platform code because the code repository is private.

However, documentation improvements for this support repository may be accepted.

Examples:
- README improvements
- issue template improvements
- wording fixes
- typo fixes

---

## Be Respectful

Please be:
- clear
- concise
- respectful
- constructive

Well-structured issues help resolve problems faster.

Thanks for helping improve Bank 🚀
